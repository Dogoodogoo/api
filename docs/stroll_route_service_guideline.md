🧬 StrollRouteService 기술 설계 명세서
본 문서는 '적합도 함수 사전 평가' 및 '유전 알고리즘'의 원리를 적용하여 산책 경로의 정밀도를 극대화하기 위한 신규 서비스 클래스(StrollRouteService)의 설계 방안을 정의합니다.

1. 개요
기존의 고정 계수 방식에서 벗어나, 백엔드 메모리 상에서 수많은 경로 후보군(Candidate Population)을 생성하고, 수학적 적합도 함수를 통해 최적의 경로를 선별한 뒤 Tmap API로 최종 검증하는 '선택형 경로 생성' 모델을 채택합니다.

2. 주요 컴포넌트 구조
2.1. 후보군 생성기 (Candidate Generator)
메커니즘: 사용자의 위치를 중심으로 각 섹터(120도) 내에서 임의의 각도와 거리를 가진 경유지 세트를 생성합니다.
다양성 확보: 한 섹터당 약 20~30개의 서로 다른 '반환점(Pivot)'과 '보정점(Anchor)' 조합을 생성하여 후보군(Population)을 구성합니다.

2.2. 적합도 함수 (Fitness Function)
각 후보 경로의 우수성을 평가하는 핵심 로직으로, 다음 가중치를 합산하여 점수를 산출합니다.
거리 정밀도 점수 (): - 공식:
굴곡률(Circuity Factor): 도심지 특성을 반영하여 1.3~1.4 배율을 적용합니다. 편차가 작을수록 높은 점수를 부여합니다.
시설물 배치 점수 ():
단순히 개수가 많은 것이 아니라, 경로 전체에 쓰레기통과 음수대가 얼마나 분산되어 배치되었는지를 엔트로피 계산 방식으로 평가합니다.
경로 기하학 점수 ():
각 경유지 간의 꺾임 각도를 계산하여, 산책에 부적합한 급격한 유턴이나 뾰족한 경로에 패널티를 부여합니다.

2.3. 엘리트 선택기 (Elite Selector)
각 섹터별로 적합도 점수가 가장 높은 상위 1개의 '엘리트 후보'를 최종 선발합니다.

3. 데이터 흐름 (Workflow)
요청 수신: 사용자의 위치 및 목표 거리 수신.
병렬 후보 생성: Java 21의 가상 스레드(Virtual Threads)를 활용하여 3개 섹션에서 동시에 수십 개의 후보 경로 세트 생성.
사전 평가 수행: Tmap 호출 없이 PostGIS와 Java 로직만으로 모든 후보의 적합도 점수 계산.
최종 후보 확정: 점수가 가장 높은 3개의 경로 세트(섹터당 1개) 추출.
Tmap 최종 검증: 확정된 3개 경로에 대해서만 Tmap 보행자 경로 API 호출 및 Polyline 획득.
결과 반환: 실제 도보 거리와 시각화 데이터를 사용자에게 제공.

4. 기대 효과
정밀도 혁신: 수치적 최적화를 거치므로 목표 거리와의 오차가 획기적으로 줄어듭니다.
안정성: 데이터가 부족한 구역에서도 여러 후보 중 그나마 가장 나은 대안을 선택하므로 시스템의 신뢰도가 상승합니다.
자원 효율성: 부적절한 경로에 대한 API 호출을 원천 차단하여 쿼터 관리 효율을 극대화합니다.
